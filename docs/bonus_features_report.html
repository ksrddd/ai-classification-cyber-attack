4. ข้อเสนอแนะ — สิ่งที่สามารถพัฒนาต่อได้

ℹ️ หมายเหตุ: ข้อเสนอแนะทั้งหมดนี้ ยังไม่ได้ทำ และอยู่นอกขอบเขต proposal — ต้องได้รับอนุมัติจากอาจารย์ก่อนดำเนินการ
⚖️ A. แก้ปัญหา Class Imbalance สำคัญมาก

ปัญหาปัจจุบัน: Bot มีแค่ 208 rows (0.07%) จาก 284,786 ทำให้ F1=0.66 ต่ำที่สุด ทุกโมเดลทำได้ไม่ดีกับ Bot

  • SMOTE (Synthetic Minority Oversampling Technique) — สร้าง synthetic samples สำหรับ class น้อย เช่น Bot, Web Attack → เพิ่ม F1 ของ minority classes
  • ADASYN (Adaptive Synthetic Sampling) — คล้าย SMOTE แต่สร้าง samples มากขึ้นในบริเวณที่ classify ยาก
  • Random Under Sampling — ลด BENIGN (2.2 ล้าน rows) ลงให้ balance กัน
  • คาดว่า F1-macro จะเพิ่มจาก 0.9456 เป็น 0.96+ สำหรับ Random Forest
📊 B. Cross-Dataset Evaluation สำคัญมาก

ปัญหาปัจจุบัน: Train และ Test ใช้ dataset เดียวกัน (CICIDS2017) — ไม่รู้ว่า model generalize ข้าม dataset ได้แค่ไหน

  • UNSW-NB15 (อยู่ใน proposal) — train บน CICIDS2017, test บน UNSW-NB15 ดู accuracy drop
  • CICIDS2018 — เป็น dataset รุ่นใหม่กว่า, attack patterns คล้ายกันแต่ traffic volume ต่างกัน
  • ผลลัพธ์นี้จะทำให้โครงงาน น่าเชื่อถือขึ้นมากในด้านวิชาการ
🧠 C. โมเดลเพิ่มเติมเพื่อการเปรียบเทียบ ปานกลาง

โมเดลที่ยังไม่ได้ทดลองและน่าสนใจ:

  • Decision Tree (CART เดี่ยว) — interpretable ที่สุด, สามารถ visualize decision rules ให้อาจารย์ดูได้, เหมาะสำหรับอธิบาย "โมเดลคิดยังไง"
  • Extra Trees — คล้าย Random Forest แต่ randomize split มากกว่า, มักเร็วกว่าและ variance ต่ำกว่า
  • Support Vector Machine (LinearSVC) — เปรียบเทียบ linear vs non-linear กับ Logistic Regression
  • K-Nearest Neighbours — non-parametric, ดูว่า instance-based learning ได้ผลต่างกันไหม
🔬 D. Feature Engineering เพิ่มเติม ปานกลาง

ตอนนี้ใช้ original 77 features จาก CICFlowMeter — สามารถสร้าง derived features:

  • Packet length ratios — Fwd/Bwd ratio, เพื่อจับ asymmetric attack patterns
  • Flow duration bins — แบ่ง short/medium/long flows เป็น categorical
  • TCP flag combinations — encode flag combination เป็น integer feature (SYN+FIN = scan pattern)
  • Port classification — well-known (0-1023), registered (1024-49151), dynamic (>49151)
🚀 E. MLOps & Deployment ระยะยาว

สำหรับ demo หรือ showcase ในอนาคต (เกิน scope โครงงานนี้มาก):

  • FastAPI REST API — POST /predict รับ JSON flow features → ส่งคืน attack type + probability
  • Docker containerization — package ทั้ง API + model เป็น container เดียว
  • MLflow Tracking — log experiments, compare model versions, model registry
  • GitHub Actions CI/CD — รัน pytest + coverage check ทุกครั้งที่ push
🔧 F. แก้ไข Coverage ที่ต่ำ (ถ้าต้องการ) ทางเลือก

Pipeline modules มี 0% coverage — ถ้าต้องการเพิ่ม:

  • Integration tests — รัน pipeline แบบ end-to-end ใน pytest โดยใช้ data/sample/synthetic_cicids.csv
  • Mock tests สำหรับ SHAP — patch shap.TreeExplainer ใน test เพื่อหลีกเลี่ยงการโหลด model จริง
  • เป้าหมาย: เพิ่ม coverage จาก 47% → 70%+ โดยไม่ต้องใช้ data จริง 880 MB
